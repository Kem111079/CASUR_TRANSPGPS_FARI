# Diseño de roles claros · CASUR Transportes GPS V5.6

## Principio funcional

- **Conductor captura.**
- **Supervisor revisa y controla.**

## Modo Conductor

Diseñado para operación en campo con baja carga visual. El usuario debe poder ejecutar el flujo sin interpretar controles técnicos:

1. Llenar datos mínimos del viaje.
2. Activar GPS.
3. Iniciar recorrido.
4. Marcar lugar si no hay lote/finca cercana.
5. Finalizar recorrido.
6. Descargar o compartir evidencia.

## Modo Supervisor

Diseñado para control operativo:

- historial local;
- folios;
- rutas anteriores;
- PDF/Excel;
- consolidado;
- capas y opciones avanzadas;
- revisión de recorridos pendientes de sincronizar.

## Nota de seguridad

Esta separación es de interfaz. La autenticación formal por usuario y rol debe implementarse en una futura app administrador con backend.
